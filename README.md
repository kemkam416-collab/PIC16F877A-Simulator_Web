## 🌍 نظرة عامة عن المشروع (Overview)
بيئة تطوير متكاملة ومحاكي تفاعلي متطور للمتحكم الدقيق **PIC16F877A** يعمل بالكامل داخل متصفح الويب دون الحاجة لتثبيت أي برامج خارجية. تم تطوير هذا المشروع ليوفر للمهندسين والطلاب تجربة برمجية واختبارية سريعة، آمنة، وعالية الأداء.

An advanced, browser-based Integrated Development Environment (IDE) and interactive simulator for the **PIC16F877A** microcontroller. Built to provide students and engineers with a seamless, zero-installation environment to write, test, and debug code efficiently.

## 📋 دليل المحاكي والقوانين البرمجية (Compiler Laws & Rules)
لكي يعمل الكود الخاص بك بنجاح على محاكي **KRIDISOFT** دون إطلاق إنذارات من "مصحح الأخطاء الذكي"، يجب الالتزام بالقواعد الهيكلية التالية:

1. **حساسية حالة الأحرف (Case Sensitivity):** جميع المسجلات والمنافذ يجب أن تُكتب بالأحرف الكبيرة (Capital Letters) مثل `PORTB` و `TRISB` وليس `portb`.
2. **نقطة الانطلاق الإلزامية:** يجب أن يحتوي الكود دائماً على الدالة الرئيسية `void main() { ... }` كحاوية أساسية للبرنامج.
3. **هيكلية قراءة المكونات الصارمة (Strict Import Syntax):** عند استدعاء شاشات الـ 7-Segment، يجب كتابة أمر الـ `import` خارج دالة `main` (في النطاق العام Global Scope)، بلواحق اسمية فريدة وبدون مسافات عشوائية حول علامات التساوي (مثل: `A=PORTB.F0;`).
4. **تسمية قطب التقطيع العشري (Decimal Point):** يتم استخدام الاختصار `DP` أو `DOT` لتعريف بن النقطة داخل أقواس الاستيراد لتحديد موضعها.

## 📊 نظام استدعاء شاشات الإظهار الساعية (7-Segment Dynamic Import)
تميز محاكي **KRIDISOFT** بنظام فريد ومبتكر يتيح للمستخدم استدعاء وتوليد شاشات الـ 7-Segment ديناميكياً ورسمها على الواجهة بمجرد كتابة أمر الاستدعاء البرمجي.

### 1. الصيغة العامة الصارمة (Strict Syntax Pattern)
يجب كتابة أمر الاستدعاء في **النطاق العام (Global Scope)** (في أعلى الملف وقبل الدالة الرئيسية)، مع الالتزام بالصيغة التالية بدقة:

```c
import <7segment_ID> {
    A = PORTX.FY;
    B = PORTX.FY;
    C = PORTX.FY;
    D = PORTX.FY;
    E = PORTX.FY;
    F = PORTX.FY;
    G = PORTX.FY;
    DP = PORTX.FY; // أو DOT = PORTX.FY;
}
حيث ID هو رقم تعريفي فريد لكل شاشة، و PORTX.FY هو اسم المنفذ والبت المربوط به قطب السقمنت المعني.

2. دعم الشاشات المتعددة (Multi-Display Support)
يمكن للمحاكي معالجة واستدعاء شاشات متعددة في نفس الوقت وبشكل مستقل تماماً عن بعضها البعض من خلال فتح بلوكات استيراد منفصلة. الشرط الوحيد هو عدم تكرار الأرجل (Pins) لتجنب تعارض الإشارات البرمجية وتداخل الأرقام.

3. مثال عملي لاستدعاء شاشتين (2x 7-Segment) في كود واحد:
الاشتقاق التالي يوضح طريقة ربط الشاشة الأولى بكامل منفذ PORTB والشاشة الثانية بكامل منفذ PORTD:

C
/* استدعاء الشاشة الأولى وتسميتها برقم 1 على بورت B */
import <7segment_1> {
    A = PORTB.F0;   B = PORTB.F1;   C = PORTB.F2;   D = PORTB.F3;
    E = PORTB.F4;   F = PORTB.F5;   G = PORTB.F6;   DP = PORTB.F7;
}

/* استدعاء الشاشة الثانية وتسميتها برقم 2 على بورت D */
import <7segment_2> {
    A = PORTD.F0;   B = PORTD.F1;   C = PORTD.F2;   D = PORTD.F3;
    E = PORTD.F4;   F = PORTD.F5;   G = PORTD.F6;   DP = PORTD.F7;
}

void main() {
    TRISB = 0x00; // تهيئة بورت B كمخرج للشاشة الأولى
    TRISD = 0x00; // تهيئة بورت D كمخرج للشاشة الثانية

    while(1) {
        PORTB = 0b00000110; // إضاءة رقم 1 على الشاشة الأولى
        PORTD = 0b01011011; // إضاءة رقم 2 على الشاشة الثانية
        delay_ms(1000);
    }
}
🛠️ كيف تعمل الدوال والأوامر المدمجة (Built-in Functions Guide)
يوفر المحاكي محركاً مدمجاً يترجم الأوامر الأساسية لعائلة الـ PIC بشكل فوري وبكفاءة عالية:

1. دوال التحكم في الاتجاه والتوجيه (I/O Direction)
TRISX = [Value];: حيث X هو اسم البورت (A, B, C, D, E).

وضع القيمة 0 يجعل المنفذ مخرجاً (Output).

وضع القيمة 1 يجعل المنفذ مدخلاً (Input).

أمثلة: TRISB = 0x00; (جعل بورت B كاملاً مخرجاً) أو TRISA = 0xFF; (جعل بورت A كاملاً مدخلاً).

2. دوال التحكم في مستويات الطاقة والبتات (Bit Operations)
PORTX.FY = [1/0];: للتحكم في بن (Pin) منفرد مشتق من بورت معُين، حيث X هو البورت و Y هو رقم البت من (0 إلى 7).

مثال: PORTB.F1 = 1; يرفع الجهد على القطب الثاني للبورت B إلى قيمة موجبة عالية (HIGH).

التعيين المباشر للبورت (Direct Port Assignment): يمكن إرسال القيم للبورت بأكمله دفعة واحدة باستخدام أنظمة عد متعددة:

النظام الثنائي (Binary): باستخدام الملحق 0b (مثال: PORTB = 0b00111111; وهي طريقة ممتازة للتحكم المباشر بإضاءة السقمنت).

النظام الستة عشري (Hexadecimal): باستخدام الملحق 0x (مثال: PORTB = 0x3F;).

النظام العشري (Decimal): كتابة الرقم مباشرة (مثال: PORTB = 63;).

3. دوال التأخير الزمني (Time Delay Functions)
Delay_ms(time); أو delay_ms(time);: دالة لتجميد تنفيذ المعالج لفترة زمنية محددة بالملي ثانية قبل الانتقال للسطر الذي يليه.

مثال: Delay_ms(600); يقوم بعمل تأخير لمدة 600 ملي ثانية (أقل من ثانية بقليل)، وهو الوقت المثالي لرؤية عدّاد الـ 7-Segment بوضوح دون تداخل مرئي.

4. دوال الحلقات والقفز (Control Flow)
يدعم المحاكي الحلقات اللانهائية التقليدية مثل while(1) { ... }.

يدعم القفز المباشر باستخدام العلامات (Labels) والأمر goto (مثال: تعريف السطر بـ loop: ثم استدعاء goto loop; في نهاية الدالة لإنشاء حلقة تكرار مستمرة).

⚡ المعمارية الداخلية وآلية المعالجة المتقدمة (Under the Hood)
الـ Web Workers وتجنب تجمد المتصفح:
المحاكاة البرمجية للمتحكمات الدقيقة تتطلب معالجة آلاف العمليات في الثانية (Sim Loop). لو تم تشغيل هذه الحلقة في خيط المعالجة الرئيسي للمتصفح، ستتوقف الصفحة عن الاستجابة تماماً. محاكي KRIDISOFT يقوم بحل هذه المعضلة عبر إنشاء "خيط معالجة معزول" في خلفية الجهاز (Web Worker) يقرأ الأكواد ويحاكي النبضات بشكل مستقل، ثم يرسل فقط تحديثات الواجهة الرسومية للخيط الرئيسي، مما يضمن أداء ثابتاً بمعدل 60 إطاراً في الثانية للواجهة مهما كانت ثقل الأكواد المكتوبة.

تقنية منع خيالات الإضاءة (Anti-Ghosting Mechanism):
في شاشات الـ 7-Segment الحقيقية والمحاكيات الضعيفة، تترك النبضات البرمجية السريعة أثراً ضوئياً خافتاً على القطع المطفأة بسبب بطء تفريغ الشحنات أو بقاء الحالة السابقة للمنفذ (State Persistence). يحتوي هذا المحاكي على نظام فحص صارم للنفاذية (Strict OFF Enforcement) يقوم بتصفية الإشارات وتفريغ حالة الأنود/الكاثود فوراً عند قراءة أي نبضة جديدة لضمان ظهور الأرقام بنقاء لوني كامل ونقي دون أي وميض خافت.

✨ الميزات الرئيسية (Key Features)
1. 🧠 محلل برمجيات ذكي ومتقدم (Static Analyzer & Validator)
فحص الهيكل البرمجي: يتحقق تلقائياً من توازن الأقواس الحاصرة {} والدائرية () قبل بدء عملية التشغيل.

التحقق من الدالة الأساسية: يتأكد من الوجود الفعلي للدالة الرئيسية void main().

مراقبة حدود العتاد الصارمة: يفحص الكود للتأكد من أن منافذ الـ PORT المكتوبة تقع في النطاق الفعلي للشريحة بين (A و E) وأن البتات تقع في النطاق الصحيح (0-7). في حال كتابة منفذ غير مدعوم في شريحة الـ PIC16F877A (مثل PORTB.F9 أو PORTZ)، يتم إيقاف تشغيل المحاكي فوراً وإرسال تقرير خطأ أحمر مفصل بالسطر لمنع انهيار النظام وتوجيه المطور للحل الصحيح.

2. 📊 محاكاة مرئية وتفاعلية (Interactive Emulation)
لوحة المكونات الذكية: محاكاة مرئية كاملة لمنافذ الـ DIP-40 الخاصة بالمتحكم مع إمكانية التفاعل مع المدخلات والمخرجات يدوياً وفي الوقت الفعلي.

شاشات إظهار متعددة: دعم كامل لتشغيل وتوليد أكثر من شاشة وتتبع حالات الإضاءة لكل واحدة على حدة بمرونة تامة وبجودة رسومية فائقة.

3. ⌨️ تجربة كتابة كود احترافية (Modern Editor Experience)
مساعد الذكاء البرمجي (IntelliSense): أداة استكمال تلقائي للنصوص والكلمات المفتاحية لسرعة كتابة الأوامر وتفادي الأخطاء الإملائية الشائعة أثناء البرمجة.

الإصلاح التلقائي (Auto-Fix): ميزة ذكية قادرة على اكتشاف وتصحيح بعض الأخطاء الهيكلية الشائعة في الشيفرة بنقرة واحدة.

التحكم بالواجهة وتناسق الألوان: واجهة ملء الشاشة (Full-Screen Layout) مبنية بـ CSS Grid متطور، مع إضاءة عالية التباين للرموز والأكواد (High-Contrast Tokens) تمنع اختفاء الرموز الدقيقة والأقواس وتجعل النص مريحاً جداً للعين عند العمل لفترات طويلة.

🛠️ التكنولوجيات المستخدمة (Built With)
HTML5 & CSS3 (Grid & Flexbox): لبناء الواجهة الرسومية وتوزيع العناصر لملء الشاشة بالكامل وتوفير تباين مميز للوضع الداكن (Dark Mode).

JavaScript (ES6+): المحرك المدمج والأساسي للمحاكاة والمحلل الاستاتيكي للأكواد (Lexer & Static Analyzer).

Web Workers API: لفصل عمليات المعالجة والمحاكاة المكثفة في الخلفية لحماية ثبات وأداء واجهة المستخدم.

SVG (Scaled Vector Graphics): للرسم الديناميكي الفوري عالي الجودة لحالة الأقطاب ومكونات الـ 7-Segment دون أي فقدان في الجودة أو البكسلة.
